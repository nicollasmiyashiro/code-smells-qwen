Modularização Insuficiente

Insufficient Modularization

📖 Definição

Modularização Insuficiente ocorre quando código que deveria estar em um módulo, pacote ou classe separada é mantido junto com código não relacionado. É o oposto de "Large Class" - aqui o problema é a falta de separação adequada.

Este smell indica que a organização do código não reflete adequadamente a estrutura do domínio ou as responsabilidades do sistema.

⚠️ Problemas Causados

  • Baixa coesão: Código não relacionado no mesmo lugar.
  • Descoberta difícil: Difícil encontrar funcionalidades específicas.
  • Reuso limitado: Não é possível importar apenas o necessário.
  • Build lento: Mudanças pequenas recompilam código desnecessário.
  • Dependências ocultas: Acoplamento não é explícito.

🔍 Como Identificar

  • Pacotes com nomes genéricos como "util", "common", "misc".
  • Classes utilitárias gigantes com métodos não relacionados.
  • Muitos imports em um arquivo (conhece muitos outros módulos).
  • Comentários separando "seções" não relacionadas.
  • Dificuldade em explicar o que um módulo faz em uma frase.

🔧 Técnicas de Refatoração

Extract Module/Packet (Extrair Módulo/Pacote)

Identifique grupos de funcionalidades relacionadas e mova-as para pacotes separados.

Extract Class (Extrair Classe)

Divida classes utilitárias grandes em classes menores e especializadas.

Introduce Module Boundaries

Defina limites claros entre módulos com dependências explícitas.

Domain-Driven Design

Organize módulos baseados no domínio do negócio, não em aspectos técnicos.
