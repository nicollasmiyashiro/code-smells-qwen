Classe Grande

Large Class / God Class

📖 Definição

Uma Classe Grande (também chamada de "God Class" ou "Classe Deus") é uma classe que assume muitas responsabilidades, contém muitos métodos e campos, e tenta fazer tudo sozinha.

Essas classes violam o Princípio da Responsabilidade Única (SRP) e se tornam pontos centrais de complexidade no sistema. São difíceis de entender, testar e manter.

⚠️ Problemas Causados

  • Complexidade alta: Difícil de entender o propósito principal.
  • Testabilidade ruim: Requer muitos mocks e setups complexos.
  • Conflitos de merge: Múltiplos desenvolvedores editam a mesma classe.
  • Acoplamento oculto: Dependências não são claras.
  • Reuso impossível: Não é possível usar apenas parte da funcionalidade.

🔍 Como Identificar

  • Classe com mais de 500-1000 linhas de código.
  • Mais de 10-15 métodos públicos.
  • Muitos campos de instância (especialmente coleções).
  • Nomes genéricos como "Manager", "Processor", "Handler".
  • Métodos que não usam todos os campos da classe.
  • Comentários separando "regiões" de funcionalidade.

✅ Java - Classes Coesas 
// Entidade de domínio
public class Pedido {
    private Long id;
    private Cliente cliente;
    private List itens;
    private StatusPedido status;
    
    public void adicionarItem(Item item) {
        itens.add(item);
    }
    
    public void removerItem(Long itemId) {
        itens.removeIf(i -> i.getId().equals(itemId));
    }
    
    public Dinheiro calcularTotal() {
        Dinheiro subtotal = calcularSubtotal();
        Dinheiro desconto = calcularDesconto();
        return subtotal.subtrair(desconto);
    }
    
    private Dinheiro calcularSubtotal() { /* ... */ }
    private Dinheiro calcularDesconto() { /* ... */ }
}

// Repository para persistência
public class PedidoRepository {

public class NotificacaoPedidoService {
    public void enviarConfirmacao(Pedido pedido) { /* ... */ }
    public void enviarCobranca(Pedido pedido) { /* ... */ }
}

// Service para validações
public class ValidacaoPedidoService {
    public void validar(Pedido pedido) { /* ... */ }
    public void validarEstoque(Pedido pedido) { /* ... */ }
}

// Service de aplicação (orquestra)
public class PedidoService {
    private PedidoRepository repository;
    private NotificacaoPedidoService notificacao;
    private ValidacaoPedidoService validacao;
    
    public void processarPedido(Pedido pedido) {
        validacao.validar(pedido);
        repository.salvar(pedido);
        notificacao.enviarConfirmacao(pedido);
    }
}

🔧 Técnicas de Refatoração

Extract Class (Extrair Classe)

Identifique grupos de campos e métodos relacionados e extraia-os para novas classes. Esta é a técnica principal para dividir classes grandes.

Extract Subclass (Extrair Subclasse)

Se parte da classe representa um subconjunto de funcionalidades, crie uma subclasse.

Extract Interface (Extrair Interface)

Para classes grandes que implementam múltiplos papéis, extraia interfaces separadas.

Separate Domain and Infrastructure

Separe lógica de domínio de código de infraestrutura (banco, email, cache).
