Campo Temporário

Temporary Field

📖 Definição

Um Campo Temporário é um campo de instância que é usado apenas em certas circunstâncias ou em partes específicas do ciclo de vida do objeto. O resto do tempo, fica vazio ou null.

Este smell indica que o campo não pertence realmente à classe - ele é relevante apenas para um contexto específico e deveria estar em outro lugar.

⚠️ Problemas Causados

  • Estado inconsistente: Objeto pode estar "incompleto".
  • Null checks everywhere: Código cheio de verificações.
  • Confusão: Quando o campo deve/m não deve estar preenchido?
  • Memória desperdiçada: Espaço alocado para dados não usados.
  • Erros de NullPointerException: Esquecer de verificar null.

🔍 Como Identificar

  • Campos que são null na maioria das vezes.
  • Verificações de null antes de usar o campo.
  • Comentários como "só é preenchido durante X".
  • Métodos que setam o campo para null após uso.
  • Campos que só fazem sentido em um estado específico.

❌ Código RUIM

❌ Java - Campo Temporário 
public class Pedido {
    private Long id;
    private Cliente cliente;
    private List itens;
    private double total;
    
    // Campo temporário - só usado durante cálculo
    private Map cacheDescontos;
    
    // Campo temporário - só usado para pedidos cancelados
    private String motivoCancelamento;
    private Date dataCancelamento;
    private Usuario canceladoPor;
    
    // Campo temporário - só usado durante importação
    private String arquivoOrigem;
    private Integer linhaArquivo;
    
    public void calcularTotal() {
        // Inicializa campo temporário
        cacheDescontos = new HashMap<>();
        
        for (Item item : itens) {
            double desconto = calcularDescontoItem(item);
            cacheDescontos.put(item.getId(), desconto);
        }
        
        // Usa cache...
        
        // Limpa campo temporário após uso
        cacheDescontos = null;
    }
    
    public void cancelar(String motivo, Usuario usuario) {
        // Preenche campos só para cancelados
        this.motivoCancelamento = motivo;
        this.dataCancelamento = new Date();
        this.canceladoPor = usuario;
    }
    
    public double getDescontoItem(Long itemId) {
        // Sempre precisa verificar null
        if (cacheDescontos != null) {
            return cacheDescontos.getOrDefault(itemId, 0.0);
        }
        return 0.0;
    }
    
    public String getMotivoCancelamento() {
        // Só faz sentido se cancelado
        if (this.status != Status.CANCELADO) {
            throw new IllegalStateException(
                "Pedido não cancelado"
            );
        }
        return motivoCancelamento;
    }
}

✅ Código BOM

✅ Java - Campos Removidos 
// Opção 1: Variável local ao invés de campo
public class Pedido {
    private Long id;
    private Cliente cliente;
    private List itens;
    
    public Dinheiro calcularTotal() {
        // Variável local - existe apenas durante o método
        Map cacheDescontos = new HashMap<>();
        
        for (Item item : itens) {
            double desconto = calcularDescontoItem(item);
            cacheDescontos.put(item.getId(), Dinheiro.valueOf(desconto));
        }
        
        return itens.stream()
            .map(item -> item.getTotal().subtrair(
                cacheDescontos.getOrDefault(item.getId(), Dinheiro.ZERO)
            ))
            .reduce(Dinheiro.ZERO, Dinheiro::adicionar);
    }
}

// Opção 2: Classe separada para estado específico
public class PedidoCancelado {
    private final Pedido pedido;
    private final String motivo;
    private final Date dataCancelamento;
    private final Usuario canceladoPor;
    
    public PedidoCancelado(Pedido pedido, String motivo, 
                           Usuario canceladoPor) {
        this.pedido = pedido;
        this.motivo = motivo;
        this.dataCancelamento = new Date();
        this.canceladoPor = canceladoPor;
    }
    
    public String getMotivo() { return motivo; }
    public Date getDataCancelamento() { return dataCancelamento; }
    public Usuario getCanceladoPor() { return canceladoPor; }
}

// Opção 3: Usar Optional para estado opcional
public class Pedido {
    private Optional cancelamento = Optional.empty();
    
    public void cancelar(String motivo, Usuario usuario) {
        this.cancelamento = Optional.of(
            new Cancelamento(motivo, usuario, new Date())
        );
    }
    
    public Optional getCancelamento() {
        return cancelamento;
    }
    
    public boolean isCancelado() {
        return cancelamento.isPresent();
    }
}

🔧 Técnicas de Refatoração

Replace Field with Local Variable

Se o campo é usado apenas em um método, transforme-o em variável local.

Extract Class (Extrair Classe)

Mova campos temporários para uma classe separada que representa o estado específico.

Introduce Null Object

Use Optional ou Null Object para representar ausência de valor explicitamente.

Replace State-Altering Field with State Pattern

Se campos mudam com o estado, use o padrão State para representar estados diferentes.
