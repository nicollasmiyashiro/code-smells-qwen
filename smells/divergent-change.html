Mudança Divergente

Divergent Change

📖 Definição

Mudança Divergente ocorre quando uma classe é frequentemente alterada por razões diferentes. Por exemplo, uma classe que muda quando há alterações em regras de negócio E quando há mudanças no banco de dados.

Este smell viola o Princípio da Responsabilidade Única (SRP): uma classe deve ter apenas um motivo para mudar.

⚠️ Problemas Causados

  • Risco de regressão: Mudança em uma área quebra outra.
  • Revisões complexas: Code reviews difíceis por misturar concerns.
  • Conflitos de merge: Times diferentes editam a mesma classe.
  • Testes frágeis: Testes falham por motivos não relacionados.
  • Dificuldade de evolução: Mudanças independentes se atrapalham.

🔍 Como Identificar

  • Histórico do git mostra mudanças por motivos diferentes.
  • Métodos agrupados por "tipo" de mudança (ex: "// Métodos de BD", "// Métodos de API").
  • Pessoas diferentes modificam a mesma classe para propósitos diferentes.
  • Releases diferentes tocam na mesma classe para features diferentes.

✅ Java - Classes Separadas por Responsabilidade 
// Regras de Negócio (muda apenas com regras)
public class CalculadoraDesconto {
    public Dinheiro calcular(Cliente cliente, Dinheiro valor) {
        if (cliente.isVip()) return valor.multiplicar(0.15);
        if (valor.ehMaiorQue(1000)) return valor.multiplicar(0.10);
        return Dinheiro.ZERO;
    }
}

public class ValidadorPedido {
    public boolean podeCancelar(Pedido pedido) {
        return pedido.getStatus() == Status.PENDENTE;
    }
}

    public Pedido buscarPorId(Long id) { /* SQL */ }

public class GeradorRelatorioVendas {

// Service de Aplicação (orquestra as demais)
public class PedidoApplicationService {
    private CalculadoraDesconto calculadora;
    private PedidoRepository repository;
    private GatewayPagamento gateway;
    private GeradorRelatorioVendas gerador;
    
    public void processarPedido(Pedido pedido) {
        Dinheiro desconto = calculadora.calcular(
            pedido.getCliente(), 
            pedido.getTotal()
        );
        pedido.aplicarDesconto(desconto);
        repository.salvar(pedido);
        gateway.processar(pedido);
    }
}

🔧 Técnicas de Refatoração

Extract Class (Extrair Classe)

Identifique grupos de métodos que mudam juntos e extraia-os para classes separadas.

Separate Concerns

Separe lógica de domínio, persistência, integração externa e apresentação em camadas/classes diferentes.

Introduce Domain Layer

Crie uma camada de domínio pura, sem dependências de infraestrutura.
