Encapsulamento Quebrado

Broken Encapsulation

📖 Definição

Encapsulamento Quebrado ocorre quando uma classe expõe seus dados internos ou detalhes de implementação, permitindo que código externo modifique seu estado diretamente.

O encapsulamento é um dos pilares da orientação a objetos. Quando quebrado, a classe perde controle sobre seu estado e invariantes, tornando-se frágil e difícil de manter.

⚠️ Problemas Causados

  • Invariantes violados: Estado interno pode ficar inconsistente.
  • Acoplamento forte: Código externo depende da implementação interna.
  • Efeitos colaterais: Mudanças externas causam bugs inesperados.
  • Refatoração difícil: Mudar implementação quebra código externo.
  • Thread safety comprometida: Múltiplos threads podem corromper estado.

🔍 Como Identificar

  • Campos públicos ou protected.
  • Getters que retornam coleções mutáveis diretamente.
  • Getters que retornam objetos mutáveis internos.
  • Setters que permitem qualquer valor sem validação.
  • Código externo modificando estado interno diretamente.
  • Constants classes com campos públicos estáticos.

❌ Código RUIM

❌ Java - Encapsulamento Quebrado 
public class Pedido {
    // Campos públicos - acesso direto permitido
    public Long id;
    public String status;
    public double total;
    
    // Coleção exposta diretamente
    public List itens = new ArrayList<>();
    
    // Getter que expõe objeto interno mutável
    public Map getMetadados() {
        return metadados; // Retorna referência direta!
    }
    
    private Map metadados = new HashMap<>();
    
    // Setter sem validação
    public void setStatus(String status) {
        this.status = status; // Qualquer valor é aceito
    }
    
    // Getter de coleção permite modificação externa
    public List getItens() {
        return itens; // Retorna lista interna!
    }
}

// Código externo pode quebrar o objeto
public class ClienteService {
    
    public void manipularPedido(Pedido pedido) {
        // Modificação direta de campo público
        pedido.total = -100; // Total negativo!
        
        // Adiciona item nulo na coleção
        pedido.itens.add(null);
        
        // Modifica mapa interno
        pedido.getMetadados().put("chave", null);
        
        // Status inválido
        pedido.setStatus("STATUS_INVALIDO_XPTO");
    }
}

✅ Código BOM

✅ Java - Encapsulamento Preservado 
public class Pedido {
    private final Long id;
    private StatusPedido status;
    private final List itens;
    private final Map metadados;
    private Dinheiro total;
    
    public Pedido(Long id) {
        this.id = id;
        this.status = StatusPedido.CRIADO;
        this.itens = new ArrayList<>();
        this.metadados = new HashMap<>();
        this.total = Dinheiro.ZERO;
    }
    
    // Getter seguro - retorna cópia ou vista imutável
    public List getItens() {
        return Collections.unmodifiableList(itens);
    }
    
    // Método de comportamento ao invés de setter
    public void adicionarItem(Item item) {
        Objects.requireNonNull(item, "Item não pode ser nulo");
        validarStatusParaAdicao();
        itens.add(item);
        recalcularTotal();
    }
    
    public void removerItem(Long itemId) {
        itens.removeIf(i -> i.getId().equals(itemId));
        recalcularTotal();
    }
    
    // Setter com validação
    public void cancelar() {
        if (status != StatusPedido.CRIADO && status != StatusPedido.PENDENTE) {
            throw new IllegalStateException(
                "Só é possível cancelar pedidos criados ou pendentes"
            );
        }
        this.status = StatusPedido.CANCELADO;
    }
    
    // Getter seguro para mapa - retorna cópia
    public Map getMetadados() {
        return new HashMap<>(metadados); // Cópia defensiva
    }
    
    public void adicionarMetadado(String chave, Object valor) {
        Objects.requireNonNull(chave, "Chave não pode ser nula");
        metadados.put(chave, valor);
    }
    
    public Dinheiro getTotal() {
        return total; // Retorna objeto imutável
    }
    
    public StatusPedido getStatus() {
        return status; // Retorna enum (imutável)
    }
    
    private void validarStatusParaAdicao() {
        if (status == StatusPedido.FINALIZADO) {
            throw new IllegalStateException(
                "Não é possível adicionar itens a pedido finalizado"
            );
        }
    }
    
    private void recalcularTotal() {
        this.total = itens.stream()
            .map(Item::getTotal)
            .reduce(Dinheiro.ZERO, Dinheiro::adicionar);
    }
}

🔧 Técnicas de Refatoração

Encapsulate Field (Encapsular Campo)

Torne campos privados e forneça getters/setters controlados.

Return Copy Instead of Reference

Retorne cópias defensivas de objetos mutáveis ao invés de referências diretas.

Replace Setter with Method

Substitua setters genéricos por métodos com nomes que expressam intenção.

Make Immutable (Tornar Imutável)

Quando possível, torne objetos imutáveis para eliminar problemas de modificação.
