Dados Espalhados

Scattered Data

📖 Definição

Dados Espalhados ocorre quando dados que deveriam estar juntos estão espalhados por várias classes. Diferente de "Data Clumps" (onde os dados viajam juntos mas não estão encapsulados), aqui os dados estão em lugares fundamentalmente diferentes.

Este smell torna difícil entender e manter o código, pois mudanças em um conceito do domínio exigem modificar múltiplas classes não relacionadas.

⚠️ Problemas Causados

  • Consistência difícil: Manter dados sincronizados é complexo.
  • Validação fragmentada: Validações em múltiplos lugares.
  • Entendimento prejudicado: Conceito único espalhado.
  • Acoplamento oculto: Classes parecem independentes mas não são.
  • Transações complexas: Atualizar todos os dados atomicamente.

🔍 Como Identificar

  • Campos com nomes similares em classes diferentes (ex: precoX, precoY).
  • Grupos de campos que sempre são atualizados juntos.
  • Comentários como "não esquecer de atualizar X quando mudar Y".
  • Métodos que recebem múltiplos parâmetros relacionados de classes diferentes.
  • Dados duplicados que devem estar sincronizados.

❌ Código RUIM

❌ Java - Dados Espalhados 
// Dados de endereço espalhados
public class Cliente {
    private String nome;
    private String email;
    // Endereço parcial
    private String rua;
    private String numero;
}

public class Pedido {
    private Long id;
    private Cliente cliente;
    // Mais dados de endereço para entrega
    private String enderecoEntregaRua;
    private String enderecoEntregaNumero;
    private String enderecoEntregaComplemento;
    private String enderecoEntregaBairro;
    private String enderecoEntregaCidade;
    private String enderecoEntregaEstado;
    private String enderecoEntregaCep;
}

public class NotaFiscal {
    private Long numero;
    // Dados de endereço para emissão
    private String enderecoClienteRua;
    private String enderecoClienteNumero;
    private String enderecoClienteBairro;
    private String enderecoClienteCidade;
    private String enderecoClienteEstado;
}

// Atualização manual e propensa a erros
public class EnderecoService {
    
    public void atualizarEndereco(
        Cliente cliente,
        String rua,
        String numero,
        String complemento,
        String bairro,
        String cidade,
        String estado,
        String cep
    ) {
        // Atualiza Cliente
        cliente.setRua(rua);
        cliente.setNumero(numero);
        
        // Atualiza todos os pedidos do cliente
        for (Pedido pedido : pedidoRepository.findByCliente(cliente)) {
            pedido.setEnderecoEntregaRua(rua);
            pedido.setEnderecoEntregaNumero(numero);
            pedido.setEnderecoEntregaComplemento(complemento);
            pedido.setEnderecoEntregaBairro(bairro);
            pedido.setEnderecoEntregaCidade(cidade);
            // ...
        }
        
        // E se esquecer de atualizar NotaFiscal?
    }
}

✅ Código BOM

✅ Java - Dados Consolidados 
// Classe de valor para Endereço
public class Endereco {
    private final String rua;
    private final String numero;
    private final String complemento;
    private final String bairro;
    private final String cidade;
    private final String estado;
    private final String cep;
    
    // Construtor, getters, equals, hashCode
    
    public String formatarCompleto() {
        return String.format("%s, %s - %s, %s - %s, %s",
            rua, numero, bairro, cidade, estado, cep);
    }
}

// Cliente com endereço consolidado
public class Cliente {
    private final String nome;
    private final String email;
    private final Endereco endereco;
    
    public Endereco getEndereco() {
        return endereco; // Retorna objeto imutável
    }
}

// Pedido referencia o endereço
public class Pedido {
    private final Long id;
    private final Cliente cliente;
    private final Endereco enderecoEntrega;
    
    public Endereco getEnderecoEntrega() {
        return enderecoEntrega != null 
            ? enderecoEntrega 
            : cliente.getEndereco();
    }
}

// NotaFiscal referencia o endereço
public class NotaFiscal {
    private final Long numero;
    private final Endereco enderecoCliente;
}

// Atualização simples e segura
public class EnderecoService {
    
    public void atualizarEndereco(Cliente cliente, Endereco novoEndereco) {
        // Atualiza em um único lugar
        // Pedidos e Notas Fiscais referenciam o endereço atualizado
        clienteRepository.atualizarEndereco(cliente, novoEndereco);
    }
}

🔧 Técnicas de Refatoração

Extract Class (Extrair Classe)

Crie uma nova classe para conter os dados espalhados. Mova todos os campos relacionados.

Move Field (Mover Campo)

Mova campos de várias classes para uma classe central que representa o conceito.

Introduce Value Object

Crie um objeto de valor imutável para representar o conceito unificado.

Replace Reference with Value

Se apropriado, substitua referências por valores copiados (snapshot).
