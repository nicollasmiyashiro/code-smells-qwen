Classe de Dados

Data Class

📖 Definição

Uma Classe de Dados é uma classe que contém apenas campos (dados) e getters/setters, sem nenhum comportamento ou lógica de negócio. É essencialmente um "saco de dados" (data bag).

Em orientação a objetos, classes devem encapsular dados E comportamento. Classes que são apenas dados violam este princípio e resultam em lógica espalhada por outras classes.

⚠️ Problemas Causados

  • Encapsulamento quebrado: Dados expostos sem proteção.
  • Lógica espalhada: Comportamento fica em classes "anêmicas".
  • Invariante não garantido: Objeto pode ficar em estado inválido.
  • Feature Envy: Outras classes manipulam os dados diretamente.
  • Domain Model Anêmico: Modelo de domínio sem comportamento.

🔍 Como Identificar

  • Classe com apenas campos privados e getters/setters públicos.
  • Nenhum método além de getters, setters, toString, equals, hashCode.
  • Classe usada apenas para transportar dados entre camadas.
  • DTOs, Entities e Models sem comportamento.
  • Validações feitas fora da classe, não dentro.

❌ Código RUIM

❌ Java - Classe de Dados 
// Classe anêmica - apenas dados
public class Pedido {
    private Long id;
    private Cliente cliente;
    private List itens;
    private double subtotal;
    private double desconto;
    private double frete;
    private double total;
    private String status;
    private Date dataCriacao;
    
    // Apenas getters e setters
    public Long getId() { return id; }
    public void setId(Long id) { this.id = id; }
    
    public Cliente getCliente() { return cliente; }
    public void setCliente(Cliente cliente) { this.cliente = cliente; }
    
    public List getItens() { return itens; }
    public void setItens(List itens) { this.itens = itens; }
    
    public double getSubtotal() { return subtotal; }
    public void setSubtotal(double subtotal) { this.subtotal = subtotal; }
    
    public double getDesconto() { return desconto; }
    public void setDesconto(double desconto) { this.desconto = desconto; }
    
    public double getFrete() { return frete; }
    public void setFrete(double frete) { this.frete = frete; }
    
    public double getTotal() { return total; }
    public void setTotal(double total) { this.total = total; }
    
    public String getStatus() { return status; }
    public void setStatus(String status) { this.status = status; }
    
    public Date getDataCriacao() { return dataCriacao; }
    public void setDataCriacao(Date dataCriacao) { this.dataCriacao = dataCriacao; }
}

// Lógica em outra classe (Feature Envy)
public class PedidoService {
    
    public void calcularTotal(Pedido pedido) {
        double subtotal = 0;
        for (Item item : pedido.getItens()) {
            subtotal += item.getPreco() * item.getQuantidade();
        }
        pedido.setSubtotal(subtotal);
        
        double desconto = 0;
        if (pedido.getCliente().isVip()) {
            desconto = subtotal * 0.15;
        }
        pedido.setDesconto(desconto);
        
        double total = subtotal - desconto + pedido.getFrete();
        pedido.setTotal(total);
    }
    
    public void adicionarItem(Pedido pedido, Item item) {
        pedido.getItens().add(item);
        calcularTotal(pedido); // Recalcula tudo
    }
}

✅ Java - Classe Rica em Comportamento 
// Classe com dados E comportamento
public class Pedido {
    private final Long id;
    private final Cliente cliente;
    private final List itens;
    private StatusPedido status;
    private final LocalDateTime dataCriacao;
    
    public Pedido(Cliente cliente) {
        this.id = UUID.randomUUID().toString();
        this.cliente = cliente;
        this.itens = new ArrayList<>();
        this.status = StatusPedido.CRIADO;
        this.dataCriacao = LocalDateTime.now();
    }
    
    public void adicionarItem(Item item) {
        validarItemAdicao(item);
        itens.add(item);
    }
    
    public void removerItem(Long itemId) {
        itens.removeIf(i -> i.getId().equals(itemId));
    }
    
    public Dinheiro calcularSubtotal() {
        return itens.stream()
            .map(Item::getTotal)
            .reduce(Dinheiro.ZERO, Dinheiro::adicionar);
    }
    
    public Dinheiro calcularDesconto() {
        if (cliente.isVip()) {
            return calcularSubtotal().multiplicar(0.15);
        }
        if (calcularSubtotal().ehMaiorQue(1000)) {
            return calcularSubtotal().multiplicar(0.10);
        }
        return Dinheiro.ZERO;
    }
    
    public Dinheiro calcularTotal() {
        return calcularSubtotal()
            .subtrair(calcularDesconto())
            .adicionar(calcularFrete());
    }
    
    public void finalizar() {
        if (itens.isEmpty()) {
            throw new IllegalStateException("Pedido sem itens");
        }
        this.status = StatusPedido.FINALIZADO;
    }
    
    public void cancelar() {
        if (status != StatusPedido.CRIADO && status != StatusPedido.PENDENTE) {
            throw new IllegalStateException("Pedido não pode ser cancelado");
        }
        this.status = StatusPedido.CANCELADO;
    }
    
    private void validarItemAdicao(Item item) {
        if (status != StatusPedido.CRIADO && status != StatusPedido.PENDENTE) {
            throw new IllegalStateException("Pedido já finalizado");
        }
    }
    
    private Dinheiro calcularFrete() {
        return cliente.calcularFrete();
    }
    
    // Getters apenas para dados necessários
    public StatusPedido getStatus() { return status; }
    public LocalDateTime getDataCriacao() { return dataCriacao; }
}

🔧 Técnicas de Refatoração

Move Method (Mover Método)

Mova métodos que operam nos dados da classe para dentro da própria classe.

Encapsulate Field (Encapsular Campo)

Torne campos privados e crie métodos que protegem invariantes ao modificar.

Replace Data Class with Domain Model

Transforme a classe de dados em um modelo de domínio rico com comportamento.

Introduce Factory Method

Crie métodos factory que garantem que o objeto seja criado em estado válido.
