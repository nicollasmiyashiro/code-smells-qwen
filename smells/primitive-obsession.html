Primitive Obsession

Primitive Obsession / Obsessão por Primitivos

📖 Definição

Primitive Obsession é o uso excessivo de tipos primitivos (String, int, double, boolean) para representar conceitos do domínio, ao invés de criar objetos especializados.

Isso resulta em código que não expressa adequadamente o domínio do problema e perde oportunidades de encapsular comportamento e validação relacionados aos dados.

⚠️ Problemas Causados

  • Falta de expressividade: Código não comunica claramente a intenção.
  • Validação duplicada: Mesmas validações repetidas em vários lugares.
  • Erros de tipo: Parâmetros do mesmo tipo podem ser trocados acidentalmente.
  • Comportamento disperso: Lógica relacionada aos dados fica espalhada.
  • Dificuldade de evolução: Mudar representação requer alterar muito código.

🔍 Como Identificar

  • Strings usadas para representar conceitos (CPF, email, telefone).
  • Validações de formato repetidas (regex para email, CPF, etc.).
  • Constantes de tipo definidas como String ou int.
  • Métodos que recebem múltiplos parâmetros do mesmo tipo primitivo.
  • Cálculos ou operações que deveriam pertencer aos dados.

✅ Código BOM

✅ Java - Value Objects 
// Value Object para Email
public class Email {
    private final String valor;
    
    public Email(String valor) {
        if (!isValidEmail(valor)) {
            throw new IllegalArgumentException("Email inválido");
        }
        this.valor = valor;
    }
    
    private boolean isValidEmail(String email) {
        return email != null && email.matches("^[A-Za-z0-9+_.-]+@(.+)$");
    }
    
    public String getValor() { return valor; }
    
    @Override
    public boolean equals(Object o) {
        if (this == o) return true;
        if (!(o instanceof Email)) return false;
        Email email = (Email) o;
        return valor.equalsIgnoreCase(email.valor);
    }
    
    @Override
    public int hashCode() {
        return valor.toLowerCase().hashCode();
    }
}

// Value Object para CPF
public class Cpf {
    private final String valor;
    
    public Cpf(String valor) {
        String numeros = valor.replaceAll("\\D", "");
        if (!isValidCpf(numeros)) {
            throw new IllegalArgumentException("CPF inválido");
        }
        this.valor = numeros;
    }
    
    private boolean isValidCpf(String cpf) {
        if (cpf == null || cpf.length() != 11) return false;
        // Lógica de validação em um único lugar
        return true;
    }
    
    public String formatado() {
        return valor.replaceFirst("(\\d{3})(\\d{3})(\\d{3})(\\d{2})", "$1.$2.$3-$4");
    }
    
    public String getValor() { return valor; }
}

// Value Object para Dinheiro
public class Dinheiro {
    private final BigDecimal valor;
    private final Currency moeda;
    
    public Dinheiro(BigDecimal valor, Currency moeda) {
        if (valor.compareTo(BigDecimal.ZERO) < 0) {
            throw new IllegalArgumentException("Valor não pode ser negativo");
        }
        this.valor = valor;
        this.moeda = moeda;
    }
    
    public Dinheiro adicionar(Dinheiro outro) {
        if (!this.moeda.equals(outro.moeda)) {
            throw new IllegalArgumentException("Moedas diferentes");
        }
        return new Dinheiro(this.valor.add(outro.valor), this.moeda);
    }
    
    public Dinheiro multiplicar(double fator) {
        return new Dinheiro(
            this.valor.multiply(BigDecimal.valueOf(fator)), 
            this.moeda
        );
    }
}

🔧 Técnicas de Refatoração

Replace Data Value with Object

Substitua um valor de dado primitivo por um objeto que contenha o valor e comportamento relacionado.

Introduce Parameter Object

Crie objetos para agrupar parâmetros relacionados, especialmente quando são do mesmo tipo primitivo.

Value Object Pattern

Crie objetos imutáveis que representam valores do domínio. Eles devem ser comparados por valor, não por identidade.

Type-Safe Enums

Substitua constantes primitivas por enums type-safe que podem ter comportamento.
