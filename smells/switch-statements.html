Switch Statements

Switch Statements / Conditional Complexity

📖 Definição

O code smell Switch Statements refere-se ao uso excessivo de instruções switch ou cadeias longas de if-else baseadas em tipos, enums ou valores específicos.

Este padrão viola o Princípio Aberto/Fechado (OCP): classes devem estar abertas para extensão mas fechadas para modificação. Cada novo tipo requer modificar o switch.

⚠️ Problemas Causados

  • Violação do OCP: Adicionar novo tipo requer modificar código existente.
  • Código duplicado: Switches similares aparecem em múltiplos lugares.
  • Complexidade ciclomática alta: Muitos caminhos de execução para testar.
  • Difícil manutenção: Mudanças exigem encontrar todos os switches relacionados.
  • Polimorfismo ignorado: Perde-se os benefícios da orientação a objetos.

🔍 Como Identificar

  • Switch baseado em tipo de objeto (instanceof + cast).
  • Switch sobre enum com muitos casos.
  • Cadeias longas de if-else if verificando tipos ou valores.
  • O mesmo switch aparece em múltiplas classes.
  • Casos do switch contêm lógica complexa que poderia ser um método próprio.

❌ Código RUIM

❌ Java - Switch por Tipo 
public class CalculadoraImposto {
    
    public double calcularImposto(Object produto, double valor) {
        if (produto instanceof Livro) {
            Livro livro = (Livro) produto;
            // Livros têm isenção parcial
            if (livro.isEducacional()) {
                return valor * 0.0;
            }
            return valor * 0.07;
        } else if (produto instanceof Alimentacao) {
            Alimentacao alimento = (Alimentacao) produto;
            // Alimentos têm alíquota reduzida
            if (alimento.isBasico()) {
                return valor * 0.04;
            }
            return valor * 0.12;
        } else if (produto instanceof Eletronico) {
            Eletronico eletronico = (Eletronico) produto;
            // Eletrônicos têm imposto alto + taxa
            double imposto = valor * 0.25;
            if (eletronico.isImportado()) {
                imposto += valor * 0.15;
            }
            return imposto;
        } else if (produto instanceof Vestuario) {
            // Vestuário varia por estado
            String estado = produto.getEstado();
            switch (estado) {
                case "SP": return valor * 0.18;
                case "RJ": return valor * 0.20;
                default: return valor * 0.22;
            }
        } else if (produto instanceof Servico) {
            return valor * 0.05;
        } else {
            // Padrão
            return valor * 0.18;
        }
    }
}

✅ Código BOM

✅ Java - Polimorfismo 
// Interface comum
public interface Tributavel {
    double calcularImposto(double valor);
}

// Implementações específicas
public class Livro implements Tributavel {
    private boolean educacional;
    
    @Override
    public double calcularImposto(double valor) {
        if (educacional) {
            return 0.0;
        }
        return valor * 0.07;
    }
}

public class Alimentacao implements Tributavel {
    private boolean basico;
    
    @Override
    public double calcularImposto(double valor) {
        if (basico) {
            return valor * 0.04;
        }
        return valor * 0.12;
    }
}

public class Eletronico implements Tributavel {
    private boolean importado;
    
    @Override
    public double calcularImposto(double valor) {
        double imposto = valor * 0.25;
        if (importado) {
            imposto += valor * 0.15;
        }
        return imposto;
    }
}

public class Vestuario implements Tributavel {
    private String estado;
    
    @Override
    public double calcularImposto(double valor) {
        return switch (estado) {
            case "SP" -> valor * 0.18;
            case "RJ" -> valor * 0.20;
            default -> valor * 0.22;
        };
    }
}

// Uso limpo e extensível
public class CalculadoraImposto {
    
    public double calcularImposto(Tributavel produto, double valor) {
        return produto.calcularImposto(valor);
    }
}

🔧 Técnicas de Refatoração

Replace Conditional with Polymorphism

Substitua condicionais baseadas em tipo por polimorfismo. Crie uma interface/classe base e implemente o comportamento em subclasses.

Replace Type Code with State/Strategy

Substitua códigos de tipo por objetos de estado ou estratégia. Permite mudar comportamento em tempo de execução.

Replace Type Code with Subclasses

Crie subclasses para cada tipo de código. O comportamento específico vai para cada subclasse.

Command Pattern

Para casos complexos, encapsule cada operação como um objeto Command. Permite adicionar novos comandos sem modificar código existente.
