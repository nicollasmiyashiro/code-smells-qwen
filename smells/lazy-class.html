Classe Preguiçosa

Lazy Class / Freeloader

📖 Definição

Uma Classe Preguiçosa (também chamada de "Freeloader") é uma classe que não faz o suficiente para justificar sua existência. Ela tem poucos métodos, poucos dados, e não representa um conceito significativo.

Este é o oposto de "Large Class". Enquanto classes grandes fazem demais, classes preguiçosas fazem de menos e deveriam ser eliminadas ou fundidas com outras.

⚠️ Problemas Causados

  • Complexidade desnecessária: Mais uma classe para entender.
  • Indireção inútil: Camada extra sem valor agregado.
  • Manutenção desperdiçada: Tempo gasto mantendo classe inútil.
  • Confusão: Desenvolvedores não entendem o propósito.
  • Performance: Objetos extras alocados sem necessidade.

🔍 Como Identificar

  • Classe com menos de 5 métodos e 3 campos.
  • Classe que só tem getters/setters e nada mais.
  • Subclasse que não adiciona nada à classe pai.
  • Classe que foi reduzida após refatorações.
  • Nomes vagos como "Helper", "Data", "Info".

❌ Código RUIM

❌ Java - Classe Preguiçosa 
// Classe que não faz nada útil sozinha
public class EnderecoInfo {
    private String rua;
    private String numero;
    private String cidade;
    private String estado;
    
    // Apenas getters e setters
    public String getRua() { return rua; }
    public void setRua(String rua) { this.rua = rua; }
    
    public String getNumero() { return numero; }
    public void setNumero(String numero) { this.numero = numero; }
    
    public String getCidade() { return cidade; }
    public void setCidade(String cidade) { this.cidade = cidade; }
    
    public String getEstado() { return estado; }
    public void setEstado(String estado) { this.estado = estado; }
}

// Classe que usa EnderecoInfo
public class Cliente {
    private String nome;
    private String email;
    private EnderecoInfo endereco;
    
    public Cliente(String nome, String email, EnderecoInfo endereco) {
        this.nome = nome;
        this.email = email;
        this.endereco = endereco;
    }
    
    // Métodos que apenas delegam
    public String getRua() {
        return endereco.getRua();
    }
    
    public String getCidade() {
        return endereco.getCidade();
    }
}

// Uso desnecessariamente complexo
public class ClienteService {
    
    public void cadastrarCliente(
        String nome,
        String email,
        String rua,
        String numero,
        String cidade,
        String estado
    ) {
        // Cria objeto intermediário desnecessário
        EnderecoInfo endereco = new EnderecoInfo();
        endereco.setRua(rua);
        endereco.setNumero(numero);
        endereco.setCidade(cidade);
        endereco.setEstado(estado);
        
        Cliente cliente = new Cliente(nome, email, endereco);
        repository.salvar(cliente);
    }
}

✅ Código BOM

✅ Java - Classe Eliminada 
// Opção 1: Mover dados para a classe principal
public class Cliente {
    private final String nome;
    private final String email;
    private final String rua;
    private final String numero;
    private final String cidade;
    private final String estado;
    
    public Cliente(String nome, String email, String rua, 
                   String numero, String cidade, String estado) {
        this.nome = nome;
        this.email = email;
        this.rua = rua;
        this.numero = numero;
        this.cidade = cidade;
        this.estado = estado;
    }
    
    public String getEnderecoFormatado() {
        return String.format("%s, %s - %s/%s", rua, numero, cidade, estado);
    }
}

// Opção 2: Usar Record (Java 16+)
public record Cliente(
    String nome,
    String email,
    String rua,
    String numero,
    String cidade,
    String estado
) {
    public String getEnderecoFormatado() {
        return String.format("%s, %s - %s/%s", rua, numero, cidade, estado);
    }
}

// Opção 3: Se Endereco tem comportamento, mantenha como Value Object
public class Endereco {
    private final String rua;
    private final String numero;
    private final String cidade;
    private final String estado;
    private final String cep;
    
    public Endereco(String rua, String numero, String cidade, 
                    String estado, String cep) {
        this.rua = rua;
        this.numero = numero;
        this.cidade = cidade;
        this.estado = estado;
        this.cep = cep;
    }
    
    public String formatar() {
        return String.format("%s, %s - %s/%s CEP: %s", 
            rua, numero, cidade, estado, cep);
    }
    
    public boolean isMesmaCidade(Endereco outro) {
        return this.cidade.equals(outro.cidade);
    }
}

// Uso simplificado
public class ClienteService {
    
    public void cadastrarCliente(
        String nome,
        String email,
        String rua,
        String numero,
        String cidade,
        String estado
    ) {
        // Direto, sem intermediário desnecessário
        Cliente cliente = new Cliente(nome, email, rua, numero, cidade, estado);
        repository.salvar(cliente);
    }
}

🔧 Técnicas de Refatoração

Inline Class (Classe Inline)

Mova todos os membros da classe preguiçosa para outra classe e elimine-a.

Collapse Hierarchy (Colapsar Hierarquia)

Se é uma subclasse vazia, remova-a e use a classe pai diretamente.

Add Behavior or Remove

Ou adicione comportamento que justifique a classe, ou elimine-a.

Use Record/Value Type

Para classes que são apenas dados, use records ou tipos de valor.
