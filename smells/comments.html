Comentários Excessivos

Comments

📖 Definição

O code smell Comentários Excessivos não significa que comentários são ruins. Significa que o código está tão pouco claro que precisa de comentários extensivos para ser entendido.

Como diz Kent Beck: "O melhor comentário é um método bem nomeado". Comentários muitas vezes são usados como curativo para código mal escrito, ao invés de refatorar o código para ser autoexplicativo.

⚠️ Problemas Causados

  • Comentários desatualizados: Código muda, comentários não.
  • Ilusão de clareza: Comentários dão falsa sensação de código claro.
  • Ruído visual: Excesso de comentários dificulta leitura.
  • Manutenção dupla: Precisa manter código E comentários sincronizados.
  • Código obscuro: Comentários mascaram complexidade do código.

🔍 Como Identificar

  • Comentários explicando o que o código faz (ao invés de porquê).
  • Blocos grandes de código comentado.
  • Comentários de cabeçalho em cada método.
  • Comentários que repetem o óbvio.
  • Comentários marcando seções ("// Região de cálculo").
  • Comentários apologéticos ("// Gambiarra temporária").

❌ Código RUIM

❌ Java - Comentários Excessivos 
public class Processador {
    
    // Método para calcular o total do pedido
    // Autor: João Silva
    // Data: 15/03/2024
    public double calcularTotal(Pedido p) {
        // Variável para armazenar o subtotal
        double subtotal = 0;
        
        // Loop para iterar sobre todos os itens
        for (int i = 0; i < p.getItens().size(); i++) {
            // Pega o item na posição i
            Item item = p.getItens().get(i);
            
            // Calcula o total do item (preço * quantidade)
            double totalItem = item.getPreco() * item.getQuantidade();
            
            // Soma ao subtotal
            subtotal = subtotal + totalItem;
        }
        
        // Verifica se tem desconto
        double desconto = 0;
        if (p.getCliente().isVip()) {
            // Cliente VIP ganha 15% de desconto
            desconto = subtotal * 0.15;
        } else if (subtotal > 1000) {
            // Pedidos acima de 1000 ganham 10%
            desconto = subtotal * 0.10;
        }
        
        // Calcula o frete baseado no estado
        double frete = 0;
        // Estado do cliente
        String estado = p.getCliente().getEstado();
        
        // SP e RJ têm frete mais barato
        if (estado.equals("SP") || estado.equals("RJ")) {
            frete = 15.0;
        } else if (estado.equals("MG") || estado.equals("ES")) {
            // MG e ES têm frete médio
            frete = 20.0;
        } else {
            // Outros estados têm frete mais caro
            frete = 30.0;
        }
        
        // Retorna o total final
        return subtotal - desconto + frete;
    }
    
    // TODO: Implementar validação de estoque
    // FIXME: Bug quando quantidade é negativa
    // HACK: Gambiarra para funcionar com sistema legado
    // XXX: Isso precisa ser refatorado urgentemente
}

✅ Código BOM

✅ Java - Código Autoexplicativo 
public class ProcessadorPedidos {
    
    public Dinheiro calcularTotalPedido(Pedido pedido) {
        Dinheiro subtotal = calcularSubtotal(pedido);
        Dinheiro desconto = calcularDesconto(pedido, subtotal);
        Dinheiro frete = calcularFrete(pedido);
        
        return subtotal.subtrair(desconto).adicionar(frete);
    }
    
    private Dinheiro calcularSubtotal(Pedido pedido) {
        return pedido.getItens().stream()
            .map(this::calcularTotalItem)
            .reduce(Dinheiro.ZERO, Dinheiro::adicionar);
    }
    
    private Dinheiro calcularTotalItem(Item item) {
        return item.getPreco().multiplicar(item.getQuantidade());
    }
    
    private Dinheiro calcularDesconto(Pedido pedido, Dinheiro subtotal) {
        if (pedido.getCliente().isVip()) {
            return subtotal.multiplicar(0.15);
        }
        if (subtotal.ehMaiorQue(new Dinheiro(1000))) {
            return subtotal.multiplicar(0.10);
        }
        return Dinheiro.ZERO;
    }
    
    private Dinheiro calcularFrete(Pedido pedido) {
        String estado = pedido.getCliente().getEstado();
        
        if (Regiao.SUDESTE.contains(estado)) {
            return new Dinheiro(15.0);
        }
        if (Regiao.SUDESTE_EXTENDIDO.contains(estado)) {
            return new Dinheiro(20.0);
        }
        return new Dinheiro(30.0);
    }
}

🔧 Técnicas de Refatoração

Introduce Explaining Variable (Introduzir Variável Explicativa)

Substitua expressões complexas por variáveis bem nomeadas que explicam seu propósito.

Rename Method (Renomear Método)

Renomeie métodos para que seu propósito seja claro sem necessidade de comentários.

Extract Method (Extrair Método)

Extraia blocos de código para métodos com nomes que explicam a intenção.

Replace Comment with Assertion

Substitua comentários sobre condições esperadas por assertions ou validações.

📊 Quando Comentários São Úteis

  • Explicar intenção (porquê): Por que esta abordagem foi escolhida.
  • Referências externas: Links para documentação, tickets, decisões.
  • Avisos de consequências: "Não remova esta linha, causa X".
  • TODOs e FIXMEs: Trabalho pendente identificado.
  • Javadoc de API pública: Documentação para consumidores da API.
  • Explicar workarounds: Soluções temporárias ou gambiarras necessárias.
