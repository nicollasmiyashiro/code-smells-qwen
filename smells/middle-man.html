Homem do Meio

Middle Man

📖 Definição

O code smell Homem do Meio ocorre quando uma classe existe apenas para delegar chamadas para outra classe. Ela não adiciona nenhum comportamento ou valor, apenas repassa mensagens.

Este é o oposto do code smell "Feature Envy". Enquanto Feature Envy move métodos para onde os dados estão, Middle Man remove classes que são apenas intermediárias desnecessárias.

⚠️ Problemas Causados

  • Complexidade desnecessária: Mais uma classe para entender.
  • Acoplamento indireto: Dependência oculta através do intermediário.
  • Manutenção extra: Mudanças na classe real exigem mudar o intermediário.
  • Performance: Chamadas extras sem benefício.
  • Confusão: Não está claro qual classe usar.

🔍 Como Identificar

  • Métodos que apenas chamam método equivalente em outro objeto.
  • Classe onde mais de 50% dos métodos são delegações.
  • Nomes como "XWrapper", "XAdapter", "XProxy" sem lógica adicional.
  • Construtor que apenas recebe e armazena referência.
  • Interface idêntica à classe delegada.

❌ Código RUIM

❌ Java - Classe Intermediária 
// Classe real com toda a lógica
public class ServicoPagamento {
    
    public boolean processar(Pagamento pagamento) {
        // Lógica complexa de processamento
        validarPagamento(pagamento);
        conectarGateway();
        enviarTransacao(pagamento);
        return confirmarTransacao();
    }
    
    public Estorno estornar(String transacaoId) {
        // Lógica de estorno
    }
    
    public Status consultarStatus(String transacaoId) {
        // Consulta status
    }
    
    public Relatorio gerarRelatorio(Date inicio, Date fim) {
        // Gera relatório
    }
}

// Homem do meio - apenas delega
public class GerenciadorPagamento {
    
    private ServicoPagamento servico;
    
    public GerenciadorPagamento(ServicoPagamento servico) {
        this.servico = servico;
    }
    
    // Apenas delega - não adiciona nada!
    public boolean processar(Pagamento pagamento) {
        return servico.processar(pagamento);
    }
    
    // Apenas delega
    public Estorno estornar(String transacaoId) {
        return servico.estornar(transacaoId);
    }
    
    // Apenas delega
    public Status consultarStatus(String transacaoId) {
        return servico.consultarStatus(transacaoId);
    }
    
    // Apenas delega
    public Relatorio gerarRelatorio(Date inicio, Date fim) {
        return servico.gerarRelatorio(inicio, fim);
    }
    
    // Getter que expõe o serviço real
    public ServicoPagamento getServico() {
        return servico;
    }
}

// Uso confuso
public class PedidoService {
    
    public void finalizarPedido(Pedido pedido) {
        // Qual usar? Gerenciador ou Servico?
        GerenciadorPagamento gerenciador = new GerenciadorPagamento(servico);
        gerenciador.processar(pedido.getPagamento());
    }
}

✅ Código BOM

✅ Java - Eliminando Intermediário 
// Usa a classe real diretamente
public class PedidoService {
    
    private ServicoPagamento servicoPagamento;
    
    // Injeção de dependência direta
    public PedidoService(ServicoPagamento servicoPagamento) {
        this.servicoPagamento = servicoPagamento;
    }
    
    public void finalizarPedido(Pedido pedido) {
        // Uso direto, sem intermediário
        servicoPagamento.processar(pedido.getPagamento());
    }
    
    public void estornarPedido(Pedido pedido) {
        servicoPagamento.estornar(pedido.getTransacaoId());
    }
}

// Se precisar de abstração, use interface
public interface ProcessadorPagamento {
    boolean processar(Pagamento pagamento);
    Estorno estornar(String transacaoId);
}

public class ServicoPagamento implements ProcessadorPagamento {
    
    @Override
    public boolean processar(Pagamento pagamento) {
        // Implementação real
    }
    
    @Override
    public Estorno estornar(String transacaoId) {
        // Implementação real
    }
}

// Se realmente precisar de wrapper, adicione valor
public class ServicoPagamentoComCache implements ProcessadorPagamento {
    
    private ProcessadorPagamento servico;
    private Cache cache;
    
    // Adiciona comportamento de cache
    @Override
    public boolean processar(Pagamento pagamento) {
        if (cache.temCache(pagamento.getId())) {
            return cache.get(pagamento.getId());
        }
        boolean resultado = servico.processar(pagamento);
        cache.put(pagamento.getId(), resultado);
        return resultado;
    }
}

🔧 Técnicas de Refatoração

Remove Middle Man (Remover Homem do Meio)

Elimine a classe intermediária e faça os clientes chamarem a classe real diretamente.

Replace Delegation with Inheritance

Se a delegação é necessária, considere usar herança ao invés de composição.

Add Value to Wrapper

Se o wrapper é necessário, adicione comportamento adicional que justifique sua existência.

Use Interface

Se precisa de abstração, use interfaces ao invés de wrappers vazios.
