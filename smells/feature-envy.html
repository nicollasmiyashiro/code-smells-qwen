Feature Envy

Feature Envy / Inveja de Feature

📖 Definição

Feature Envy ocorre quando um método em uma classe usa mais dados ou métodos de outra classe do que da sua própria classe. É como se o método "invejasse" os dados da outra classe.

Este smell indica que o método está mais relacionado à classe "invejada" do que à classe onde está definido, violando o princípio de que dados e comportamento devem estar juntos.

⚠️ Problemas Causados

  • Alto acoplamento: Método depende fortemente de outra classe.
  • Baixa coesão: Comportamento não está com seus dados.
  • Violação de encapsulamento: Dados são expostos via getters.
  • Fragilidade: Mudanças na classe "invejada" quebram o método.
  • Dificuldade de teste: Requer mocks complexos da outra classe.

🔍 Como Identificar

  • Método chama muitos getters de outro objeto.
  • Mais chamadas a métodos de outra classe do que da própria.
  • Parâmetro do método é acessado mais que this.
  • Cálculos que usam apenas dados de outro objeto.
  • Método que poderia ser nomeado como parte da outra classe.

❌ Código RUIM

❌ Java - Feature Envy 
// Classe com dados
public class Pedido {
    private List itens;
    private Cliente cliente;
    private double subtotal;
    private double desconto;
    private double frete;
    
    // Apenas getters e setters
    public List getItens() { return itens; }
    public Cliente getCliente() { return cliente; }
    public double getSubtotal() { return subtotal; }
    public double getDesconto() { return desconto; }
    public double getFrete() { return frete; }
    // ... setters
}

// Classe que "inveja" Pedido
public class RelatorioService {
    
    public double calcularTotal(Pedido pedido) {
        // Usa apenas dados do pedido, nada do serviço
        double subtotal = 0;
        for (Item item : pedido.getItens()) {
            subtotal += item.getPreco() * item.getQuantidade();
        }
        
        double desconto = 0;
        if (pedido.getCliente().isVip()) {
            desconto = subtotal * 0.15;
        } else if (subtotal > 1000) {
            desconto = subtotal * 0.10;
        }
        
        double frete = calcularFrete(pedido.getCliente());
        
        double total = subtotal - desconto + frete;
        double imposto = total * 0.18;
        
        return total + imposto;
    }
    
    public double calcularFrete(Cliente cliente) {
        // Também inveja Cliente
        String estado = cliente.getEstado();
        return switch (estado) {
            case "SP", "RJ" -> 15.0;
            case "MG", "ES" -> 20.0;
            default -> 30.0;
        };
    }
    
    public String gerarResumo(Pedido pedido) {
        // Mais inveja de Pedido
        StringBuilder sb = new StringBuilder();
        sb.append("Pedido de: ").append(pedido.getCliente().getNome());
        sb.append("\nItens: ").append(pedido.getItens().size());
        sb.append("\nSubtotal: R$ ").append(pedido.getSubtotal());
        sb.append("\nDesconto: R$ ").append(pedido.getDesconto());
        return sb.toString();
    }
}

✅ Código BOM

✅ Java - Comportamento na Classe Certa 
// Classe com dados E comportamento
public class Pedido {
    private List itens;
    private Cliente cliente;
    
    public double calcularSubtotal() {
        return itens.stream()
            .mapToDouble(i -> i.getPreco() * i.getQuantidade())
            .sum();
    }
    
    public double calcularDesconto() {
        double subtotal = calcularSubtotal();
        if (cliente.isVip()) {
            return subtotal * 0.15;
        } else if (subtotal > 1000) {
            return subtotal * 0.10;
        } else if (subtotal > 500) {
            return subtotal * 0.05;
        }
        return 0;
    }
    
    public double calcularFrete() {
        return cliente.calcularFrete();
    }
    
    public double calcularTotal() {
        double subtotal = calcularSubtotal();
        double desconto = calcularDesconto();
        double frete = calcularFrete();
        double totalParcial = subtotal - desconto + frete;
        return totalParcial + (totalParcial * 0.18);
    }
    
    public String gerarResumo() {
        return String.format(
            "Pedido de: %s\nItens: %d\nTotal: R$ %.2f",
            cliente.getNome(),
            itens.size(),
            calcularTotal()
        );
    }
}

// Cliente também tem seu comportamento

            default -> 30.0;
        };
    }
}

🔧 Técnicas de Refatoração

Move Method (Mover Método)

Mova o método que tem feature envy para a classe que ele mais usa. Esta é a refatoração principal para este smell.

Hide Delegate (Esconder Delegado)

Crie métodos na classe intermediária para esconder o acesso direto aos objetos delegados.

Extract Method + Move Method

Extraia a parte do método que tem envy e mova para a classe apropriada.
