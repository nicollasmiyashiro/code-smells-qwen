Herança Paralela

Parallel Inheritance Hierarchies

📖 Definição

Herança Paralela ocorre quando criar uma subclasse em uma hierarquia exige criar uma subclasse correspondente em outra hierarquia. As duas hierarquias crescem em paralelo, sempre espelhando uma à outra.

Este smell indica que as duas hierarquias deveriam estar mais próximas - talvez uma deveria referenciar a outra, ou deveriam ser combinadas.

⚠️ Problemas Causados

  • Duplicação de estrutura: Duas hierarquias para o mesmo conceito.
  • Acoplamento forte: Hierarquias sempre mudam juntas.
  • Esquecimento fácil: Criar uma subclasse e esquecer a correspondente.
  • Explosão de classes: Número de classes multiplica rapidamente.
  • Manutenção dobrada: Mudanças em duas hierarquias.

🔍 Como Identificar

  • Prefixos/sufixos correspondentes (ClientePF/ClientePJ e ContaPF/ContaPJ).
  • Fábricas que criam pares de objetos relacionados.
  • Switch/if-else baseado no tipo que cria objetos da outra hierarquia.
  • Comentários como "não esqueça de criar X correspondente".
  • Hierarquias com o mesmo número de subclasses.

❌ Código RUIM

❌ Java - Hierarquias Paralelas 
// Hierarquia de Funcionários
public abstract class Funcionario {
    protected String nome;
    protected double salarioBase;
    
    public abstract double calcularBonus();
}

public class FuncionarioCLT extends Funcionario {
    @Override
    public double calcularBonus() {
        return salarioBase * 0.5; // 50% do salário
    }
}

public class FuncionarioPJ extends Funcionario {
    @Override
    public double calcularBonus() {
        return salarioBase * 0.1; // 10% do salário
    }
}

public class Estagiario extends Funcionario {
    @Override
    public double calcularBonus() {
        return 0; // Sem bônus
    }
}

// Hierarquia PARALELA de Benefícios
public abstract class BeneficioFuncionario {
    protected Funcionario funcionario;
    
    public abstract List getBeneficios();
}

public class BeneficioCLT extends BeneficioFuncionario {
    @Override
    public List getBeneficios() {
        return Arrays.asList("VT", "VR", "Plano de Saúde", "INSS");
    }
}

public class BeneficioPJ extends BeneficioFuncionario {
    @Override
    public List getBeneficios() {
        return Arrays.asList("Sem benefícios obrigatórios");
    }
}

public class BeneficioEstagiario extends BeneficioFuncionario {
    @Override
    public List getBeneficios() {
        return Arrays.asList("VT", "Seguro de Vida");
    }
}

// Fábrica que espelha as hierarquias
public class BeneficioFactory {
    
    public static BeneficioFuncionario criar(Funcionario funcionario) {
        if (funcionario instanceof FuncionarioCLT) {
            return new BeneficioCLT();
        } else if (funcionario instanceof FuncionarioPJ) {
            return new BeneficioPJ();
        } else if (funcionario instanceof Estagiario) {
            return new BeneficioEstagiario();
        }
        throw new IllegalArgumentException("Tipo desconhecido");
    }
}

✅ Java - Hierarquia Unificada 
// Tipo de funcionário como enum
public enum TipoFuncionario {
    CLT(0.5, Arrays.asList("VT", "VR", "Plano de Saúde", "INSS")),
    PJ(0.1, Arrays.asList("Sem benefícios obrigatórios")),
    ESTAGIARIO(0, Arrays.asList("VT", "Seguro de Vida"));
    
    private final double bonusPercentual;
    private final List beneficios;
    
    TipoFuncionario(double bonus, List beneficios) {
        this.bonusPercentual = bonus;
        this.beneficios = beneficios;
    }
    
    public double calcularBonus(double salarioBase) {
        return salarioBase * bonusPercentual;
    }
    
    public List getBeneficios() {
        return new ArrayList<>(beneficios);
    }
}

// Funcionário simples com tipo
public class Funcionario {
    private final String nome;
    private final double salarioBase;
    private final TipoFuncionario tipo;
    
    public Funcionario(String nome, double salarioBase, TipoFuncionario tipo) {
        this.nome = nome;
        this.salarioBase = salarioBase;
        this.tipo = tipo;
    }
    
    public double calcularBonus() {
        return tipo.calcularBonus(salarioBase);
    }
    
    public List getBeneficios() {
        return tipo.getBeneficios();
    }
    
    public TipoFuncionario getTipo() {
        return tipo;
    }
}

// Uso simples
public class RHService {
    
    public void processarFolha(Funcionario funcionario) {
        double bonus = funcionario.calcularBonus();
        List beneficios = funcionario.getBeneficios();
        
        System.out.println(funcionario.getNome() + ":");
        System.out.println("  Bônus: R$ " + bonus);
        System.out.println("  Benefícios: " + beneficios);
    }
}

🔧 Técnicas de Refatoração

Move Method (Mover Método)

Mova métodos de uma hierarquia para a outra, consolidando o comportamento.

Collapse Hierarchy (Colapsar Hierarquia)

Combine as duas hierarquias em uma única, usando composição ou enum.

Replace Type Code with Enum

Substitua hierarquias paralelas por um enum com comportamento.

Introduce Association

Faça uma hierarquia referenciar a outra ao invés de espelhá-la.
