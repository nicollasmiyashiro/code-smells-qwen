Muitos Argumentos

Many Arguments / Long Parameter List

📖 Definição

O code smell Muitos Argumentos ocorre quando um método recebe uma quantidade excessiva de parâmetros. Não existe um número mágico, mas geralmente mais de 3-4 parâmetros já é um sinal de alerta.

Listas longas de parâmetros tornam o código difícil de ler, entender e manter. Além disso, indicam que o método pode estar fazendo demais ou que os dados deveriam estar encapsulados em um objeto.

⚠️ Problemas Causados

  • Dificuldade de uso: Chamar o método requer lembrar a ordem e significado de muitos parâmetros.
  • Acoplamento excessivo: O método depende de muitos dados externos.
  • Refatoração difícil: Adicionar um novo parâmetro requer mudar todas as chamadas.
  • Código confuso: Chamadas com muitos argumentos são difíceis de ler.
  • Violação de encapsulamento: Dados relacionados não estão agrupados.

🔍 Como Identificar

  • Métodos com mais de 4 parâmetros.
  • Parâmetros que sempre aparecem juntos em várias chamadas.
  • Grupos de parâmetros que representam partes de um conceito maior.
  • Parâmetros do mesmo tipo que podem ser confundidos na ordem.
  • Métodos que recebem objetos inteiros mas usam apenas alguns campos.

❌ Código RUIM

❌ Java - Muitos Parâmetros 
public class CadastroCliente {
    
    public void cadastrarCliente(
        String nome,
        String sobrenome,
        String email,
        String telefone,
        String cpf,
        String rua,
        String numero,
        String complemento,
        String bairro,
        String cidade,
        String estado,
        String cep,
        LocalDate dataNascimento,
        String sexo,
        String estadoCivil
    ) {
        // Validar dados
        if (nome == null || nome.isEmpty()) {
            throw new IllegalArgumentException("Nome obrigatório");
        }
        
        // Criar endereço
        String enderecoCompleto = rua + ", " + numero;
        if (complemento != null) {
            enderecoCompleto += " - " + complemento;
        }
        enderecoCompleto += "\n" + bairro;
        enderecoCompleto += "\n" + cidade + " - " + estado;
        enderecoCompleto += "\nCEP: " + cep;
        
        // Calcular idade
        int idade = Period.between(dataNascimento, LocalDate.now()).getYears();
        
        // Salvar no banco
        Cliente cliente = new Cliente();
        cliente.setNome(nome + " " + sobrenome);
        cliente.setEmail(email);
        cliente.setTelefone(telefone);
        cliente.setCpf(cpf);
        cliente.setEndereco(enderecoCompleto);
        cliente.setDataNascimento(dataNascimento);
        cliente.setIdade(idade);
        cliente.setSexo(sexo);
        cliente.setEstadoCivil(estadoCivil);
        
        repository.salvar(cliente);
    }
}

// Classes de valor bem definidas
public class Endereco {
    private final String rua;
    private final String numero;
    private final String complemento;
    private final String bairro;
    private final String cidade;
    private final String estado;
    private final String cep;
    
    // Construtor, getters e métodos úteis
    public String getEnderecoCompleto() {
        StringBuilder sb = new StringBuilder(rua + ", " + numero);
        if (complemento != null) {
            sb.append(" - ").append(complemento);
        }
        sb.append("\n").append(bairro);
        sb.append("\n").append(cidade).append(" - ").append(estado);
        sb.append("\nCEP: ").append(cep);
        return sb.toString();
    }
}

public class CadastroCliente {
    
    public void cadastrarCliente(Cliente cliente) {
        validarCliente(cliente);
        repository.salvar(cliente);
    }
    
    private void validarCliente(Cliente cliente) {
        if (cliente.getNome() == null || cliente.getNome().isEmpty()) {
            throw new IllegalArgumentException("Nome obrigatório");
        }
        if (cliente.getEmail() == null || !cliente.getEmail().contains("@")) {
            throw new IllegalArgumentException("Email inválido");
        }
    }
}

🔧 Técnicas de Refatoração

Introduce Parameter Object (Introduzir Objeto de Parâmetro)

Crie uma classe para agrupar parâmetros que estão relacionados. Substitua os parâmetros individuais pelo novo objeto.

Preserve Whole Object (Preservar Objeto Inteiro)

Ao invés de passar valores individuais extraídos de um objeto, passe o objeto inteiro e extraia os valores dentro do método.

Replace Parameter with Method (Substituir Parâmetro por Método)

Se um parâmetro pode ser calculado dentro do método, remova-o e faça o cálculo internamente.

Builder Pattern

Para objetos com muitos campos opcionais, use o padrão Builder para construir o objeto de forma fluente e legível.
