Cirurgia de Escopeta

Shotgun Surgery

📖 Definição

Cirurgia de Escopeta ocorre quando uma mudança simples no sistema requer modificar muitas classes diferentes. Como um tiro de escopeta, a mudança "espalha" por todo o código.

Este é o oposto de "Divergent Change". Enquanto Divergent Change é uma classe que muda por muitos motivos, Shotgun Surgery é uma mudança que afeta muitas classes.

⚠️ Problemas Causados

  • Alto risco: Fácil esquecer de atualizar alguma classe.
  • Manutenção cara: Cada mudança exige muito trabalho.
  • Bugs frequentes: Classes esquecidas causam erros.
  • Desenvolvimento lento: Medo de mudar código.
  • Testes frágeis: Muitos testes quebram com cada mudança.

🔍 Como Identificar

  • Commit que modifica 10+ arquivos para uma feature pequena.
  • Comentários como "lembre-se de atualizar X, Y e Z também".
  • Constants duplicadas em várias classes.
  • Mesma mudança de lógica em múltiplos lugares.
  • Revisores de PR sempre pedindo "atualizou a classe X?".

❌ Código RUIM

❌ Java - Mudança Espalhada 
// Constante duplicada em várias classes
public class RelatorioPDF {
    private static final String IMPOSTO_ALIQUOTA = "18%";
    
    public void gerar() {
        // Usa IMPOSTO_ALIQUOTA
    }
}

public class RelatorioExcel {
    private static final String IMPOSTO_ALIQUOTA = "18%";
    
    public void gerar() {
        // Usa IMPOSTO_ALIQUOTA
    }
}

public class CalculadoraImposto {
    private static final double IMPOSTO_ALIQUOTA = 0.18;
    
    public double calcular(double valor) {
        return valor * IMPOSTO_ALIQUOTA;
    }
}

public class NotaFiscal {
    private static final double ALIQUOTA_IMPOSTO = 0.18;
    
    public double getValorImposto() {
        return getTotal() * ALIQUOTA_IMPOSTO;
    }
}

public class SimuladorPreco {
    private static final double TAXA_IMPOSTO = 0.18;
    
    public double simular(double custo) {
        return custo + (custo * TAXA_IMPOSTO);
    }
}

// Para mudar a alíquota de 18% para 20%:
// 1. Editar RelatorioPDF
// 2. Editar RelatorioExcel
// 3. Editar CalculadoraImposto
// 4. Editar NotaFiscal
// 5. Editar SimuladorPreco
// 6. Torcer para não esquecer nenhum!

✅ Código BOM

✅ Java - Mudança Centralizada 
// Configuração centralizada
public class ConfiguracaoTributaria {
    
    private static ConfiguracaoTributaria instancia;
    private double aliquotaIcms = 0.18;
    private double aliquotaIpi = 0.05;
    private double aliquotaPis = 0.0065;
    private double aliquotaCofins = 0.03;
    
    private ConfiguracaoTributaria() {}
    
    public static ConfiguracaoTributaria getInstancia() {
        if (instancia == null) {
            instancia = new ConfiguracaoTributaria();
        }
        return instancia;
    }
    
    public double getAliquotaIcms() { return aliquotaIcms; }
    public double getAliquotaIpi() { return aliquotaIpi; }
    public double getAliquotaPis() { return aliquotaPis; }
    public double getAliquotaCofins() { return aliquotaCofins; }
    
    public double getAliquotaTotal() {
        return aliquotaIcms + aliquotaIpi + aliquotaPis + aliquotaCofins;
    }
    
    // Pode carregar de arquivo de configuração
    public void carregarDeArquivo(String path) {
        // Carrega configurações
    }
}

// Classes usam configuração centralizada
public class CalculadoraImposto {
    
    public Dinheiro calcular(Dinheiro valor) {
        double aliquota = ConfiguracaoTributaria
            .getInstancia()
            .getAliquotaTotal();
        return valor.multiplicar(aliquota);
    }
}

public class NotaFiscal {
    
    public Dinheiro getValorImposto() {
        return getTotal().multiplicar(
            ConfiguracaoTributaria.getInstancia().getAliquotaTotal()
        );
    }
}

// Para mudar a alíquota:
// 1. Editar apenas ConfiguracaoTributaria
// 2. Ou carregar novo arquivo de configuração
// 3. Todas as classes usam automaticamente o novo valor!

🔧 Técnicas de Refatoração

Consolidate Duplicate Conditional Fragments

Mova lógica duplicada para um único método e chame de todos os lugares.

Introduce Configuration Class

Centralize valores e configurações em uma classe dedicada.

Replace Magic Number with Constant

Substitua valores literais por constantes centralizadas.

Introduce Strategy Pattern

Para comportamentos que variam, use Strategy para centralizar a variação.
