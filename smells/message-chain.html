Cadeia de Mensagens

Message Chain / Train Wreck

📖 Definição

Uma Cadeia de Mensagens (também chamada de "Train Wreck") ocorre quando você vê uma longa sequência de chamadas de métodos encadeados, como objeto.getA().getB().getC().getD().

Este padrão viola a Lei de Demeter (Princípio do Menor Conhecimento): um objeto deve conversar apenas com seus amigos diretos, não com os amigos de seus amigos.

⚠️ Problemas Causados

  • Acoplamento profundo: Código conhece estrutura interna de objetos.
  • Fragilidade: Mudança em qualquer elo quebra toda a cadeia.
  • Difícil teste: Requer mockar toda a cadeia de objetos.
  • NullSafety: Cada elo pode ser null, exigindo verificações.
  • Violação de encapsulamento: Atravessa múltiplas camadas.

🔍 Como Identificar

  • Cadeias com 3+ chamadas de método (getA().getB().getC()).
  • Múltiplos pontos (.) em uma única linha de código.
  • Código que navega por várias camadas do modelo.
  • Verificações de null em cascata.
  • Conhecimento de estrutura interna de outras classes.

❌ Código RUIM

❌ Java - Cadeia Longa 
public class PedidoService {
    
    public String getCidadeEntrega(Pedido pedido) {
        // Cadeia longa - Train Wreck!
        return pedido
            .getCliente()
            .getEndereco()
            .getCidade()
            .getNome();
    }
    
    public String getNomeContato(Pedido pedido) {
        // Outra cadeia longa
        return pedido
            .getCliente()
            .getContato()
            .getNomeCompleto();
    }
    
    public double getDesconto(Pedido pedido) {
        // Cadeia com verificação de null
        if (pedido != null 
            && pedido.getCliente() != null
            && pedido.getCliente().getCategoria() != null) {
            
            return pedido
                .getCliente()
                .getCategoria()
                .getDesconto();
        }
        return 0;
    }
    
    public boolean isEntregaEspecial(Pedido pedido) {
        // Cadeia muito longa
        return pedido
            .getCliente()
            .getEndereco()
            .getCidade()
            .getEstado()
            .getRegiao()
            .isEspecial();
    }
}

✅ Código BOM

✅ Java - Delegação Adequada 
// Pedido com métodos delegados
public class Pedido {
    private Cliente cliente;
    
    // Delega para Cliente
    public String getCidadeEntrega() {
        return cliente != null 
            ? cliente.getCidadeEntrega() 
            : null;
    }
    
    // Delega para Cliente
    public String getNomeContato() {
        return cliente != null 
            ? cliente.getNomeContato() 
            : null;
    }
    
    // Delega para Cliente
    public double getDesconto() {
        return cliente != null 
            ? cliente.getDesconto() 
            : 0;
    }
    
    // Delega para Cliente
    public boolean isEntregaEspecial() {
        return cliente != null 
            ? cliente.isEntregaEspecial() 
            : false;
    }
}

// Cliente também delega
public class Cliente {
    private Endereco endereco;
    private Contato contato;
    private Categoria categoria;
    
    public String getCidadeEntrega() {
        return endereco != null 
            ? endereco.getCidadeNome() 
            : null;
    }
    
    public String getNomeContato() {
        return contato != null 
            ? contato.getNomeCompleto() 
            : null;
    }
    
    public double getDesconto() {
        return categoria != null 
            ? categoria.getDesconto() 
            : 0;
    }
    
    public boolean isEntregaEspecial() {
        return endereco != null 
            && endereco.getEstado() != null
            && endereco.getEstado().getRegiao().isEspecial();
    }
}

🔧 Técnicas de Refatoração

Hide Delegate (Esconder Delegado)

Crie métodos na classe intermediária que escondem a chamada ao objeto delegado.

Extract Method (Extrair Método)

Extraia a cadeia para um método com nome que expressa a intenção.

Move Method (Mover Método)

Mova o método que usa a cadeia para mais perto do final da cadeia.

Introduce Null Object

Substitua verificações de null por objetos Null que têm comportamento padrão.
