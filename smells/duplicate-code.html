Código Duplicado

Duplicate Code

📖 Definição

Código Duplicado é considerado o code smell mais grave. Ocorre quando a mesma estrutura de código aparece em múltiplos lugares no sistema.

A duplicação viola o princípio DRY (Don't Repeat Yourself) e é a raiz de muitos problemas de manutenção. Como diz Kent Beck: "Toda vez que você se repetir, é uma oportunidade de extrair um método".

⚠️ Problemas Causados

  • Manutenção multiplicada: Mudanças devem ser feitas em vários lugares.
  • Bugs inconsistentes: Correção em um local pode não ser aplicada em outros.
  • Código inchado: Base de código maior que o necessário.
  • Testes duplicados: Mesma lógica testada múltiplas vezes.
  • Dificuldade de evolução: Melhorias não se propagam automaticamente.

🔍 Como Identificar

  • Copiar e colar código com pequenas variações.
  • Mesma lógica em métodos diferentes.
  • Blocos de código similares em classes diferentes.
  • Comentários que explicam código repetido.
  • Métodos que diferem apenas por nomes de variáveis ou tipos.

✅ Java - Código Consolidado 
// Utilitário centralizado
public class Formatador {
    
    public static String formatar(String... partes) {
        if (partes == null || partes.length == 0) {
            return "";
        }
        
        return Arrays.stream(partes)
            .map(p -> p != null ? p.trim() : "")
            .filter(p -> !p.isEmpty())
            .collect(Collectors.joining(" "));
    }
    
    public static String formatarComSeparador(String separador, String... partes) {
        if (partes == null || partes.length == 0) {
            return "";
        }
        
        return Arrays.stream(partes)
            .map(p -> p != null ? p.trim() : "")
            .filter(p -> !p.isEmpty())
            .collect(Collectors.joining(separador));
    }
    
    public static  String formatarObjeto(T obj, Function extrator) {
        if (obj == null) {
            return "";
        }
        return formatar(extrator.apply(obj));
    }
}

// Classes de relatório limpas
public class RelatorioCliente {
    
    public String formatarNomeCompleto(Cliente cliente) {
        return Formatador.formatarObjeto(cliente, c -> 
            new String[]{c.getPrimeiroNome(), c.getSobrenome()}
        );
    }
    
    public String formatarEndereco(Cliente cliente) {
        return Formatador.formatarComSeparador(", ", 
            cliente != null ? cliente.getRua() : null,
            cliente != null ? cliente.getCidade() : null
        );
    }
}

public class RelatorioProduto {
    
    public String formatarNomeCompleto(Produto produto) {
        return Formatador.formatarComSeparador(" - ", 
            produto != null ? produto.getNome() : null,
            produto != null ? produto.getMarca() : null
        );
    }
}

public class RelatorioPedido {
    
    public String formatarNumero(Pedido pedido) {
        return Formatador.formatarComSeparador("-", 
            pedido != null ? pedido.getPrefixo() : null,
            pedido != null ? pedido.getNumero() : null
        );
    }
}

🔧 Técnicas de Refatoração

Extract Method (Extrair Método)

Extraia o código duplicado para um método único e substitua as duplicações por chamadas a esse método.

Pull Up Method (Puxar Método para Cima)

Se o código duplicado está em subclasses, mova-o para a classe base.

Form Template Method

Crie um método template na classe base com a estrutura comum e deixe as subclasses implementarem as partes variáveis.

Introduce Utility Class

Para código duplicado em classes não relacionadas, crie uma classe utilitária com métodos estáticos.
