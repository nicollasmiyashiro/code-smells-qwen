Código Morto

Dead Code

📖 Definição

Código Morto é qualquer código que não é mais executado ou necessário, mas permanece na base de código. Isso inclui métodos não chamados, variáveis não usadas, classes órfãs e código comentado.

Código morto é como lixo acumulado: ocupa espaço, causa confusão e torna a manutenção mais difícil. Se não está sendo usado, deve ser removido.

⚠️ Problemas Causados

  • Confusão: Desenvolvedores não sabem se o código é necessário.
  • Manutenção desperdiçada: Tempo gasto entendendo código inútil.
  • Build mais lento: Código extra para compilar e analisar.
  • Falsos positivos: Ferramentas alertam sobre código que parece bug.
  • Versionamento inchado: Histórico do git com código desnecessário.

🔍 Como Identificar

  • Métodos privados nunca chamados.
  • Variáveis atribuídas mas nunca lidas.
  • Classes não referenciadas em nenhum lugar.
  • Código comentado (blocos grandes de // ou /* */).
  • Parâmetros de método não utilizados.
  • Imports não usados.
  • Código após return, throw ou break incondicional.

❌ Código RUIM

❌ Java 
public class ProcessadorPagamento {

    // Variável nunca usada
    private static final String VERSAO = "2.0";
    private double taxaProcessamento = 0.029;

    // Método antigo - não remover, pode ser útil depois
    /*
    public double calcularTaxaAntiga(double valor) {
        return valor * 0.035;
    }
    */

    // Método privado nunca chamado
    private void logProcessamento(String mensagem) {
        System.out.println("[LOG] " + mensagem);
    }

    // Parâmetro não usado
    public boolean processar(double valor, String moeda) {
        // moeda não é usada
        if (valor <= 0) {
            return false;
        }

        double taxa = taxaProcessamento;

        // Código inalcançável
        return valor * taxa > 0;
        System.out.println("Processado"); // Nunca executado
    }

    // Método público nunca chamado externamente
    public void metodoLegado() {
        // Foi usado apenas em testes antigos
        System.out.println("Legado");
    }
}

🔧 Técnicas de Refatoração

Remove Dead Code (Remover Código Morto)

Simplesmente delete o código não usado. Se precisar no futuro, use controle de versão (git) para recuperar.

Remove Unused Parameter (Remover Parâmetro Não Usado)

Remova parâmetros que não são usados no corpo do método. Atualize todas as chamadas.

Remove Unused Field (Remover Campo Não Usado)

Delete campos que não são lidos em nenhum lugar da classe.

Remove Unused Method (Remover Método Não Usado)

Elimine métodos que não são chamados em nenhum lugar do sistema.
