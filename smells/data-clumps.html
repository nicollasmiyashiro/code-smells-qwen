Data Clumps / Grupos de Dados

📖 Definição

Data Clumps ocorrem quando grupos de dados aparecem juntos repetidamente em várias partes do código, mas não estão encapsulados em uma estrutura própria.

Esses "agrupamentos" de dados são frequentemente passados como parâmetros juntos, armazenados em variáveis com nomes similares, ou aparecem como campos relacionados em classes diferentes.

⚠️ Problemas Causados

  • Código duplicado: A mesma estrutura de dados é recriada em vários lugares.
  • Acoplamento acidental: Dados relacionados não têm uma unidade clara.
  • Dificuldade de mudança: Alterar a estrutura requer modificar múltiplos locais.
  • Perda de comportamento: Dados sem classe não podem ter métodos relacionados.
  • Validação repetida: Validações dos dados são duplicadas.

🔍 Como Identificar

  • Variáveis com prefixos/sufixos similares (enderecoRua, enderecoNumero, enderecoCidade).
  • Parâmetros que sempre aparecem juntos em múltiplos métodos.
  • Campos similares em classes diferentes (nome, sobrenome, email em várias classes).
  • Arrays ou listas usadas para agrupar dados relacionados.
  • Comentários descrevendo relacionamentos entre campos separados.

✅ Código BOM

✅ Java - Classes de Valor 
// Classe para dados do cliente
public class Cliente {
    private final String nome;
    private final String email;
    private final String telefone;
    
    public Cliente(String nome, String email, String telefone) {
        this.nome = nome;
        this.email = email;
        this.telefone = telefone;
    }
    
    // Getters
    public String getNome() { return nome; }
    public String getEmail() { return email; }
    public String getTelefone() { return telefone; }
}

// Classe para endereço
public class Endereco {
    private final String rua;
    private final String numero;
    private final String complemento;
    private final String bairro;
    private final String cidade;
    private final String estado;
    private final String cep;
    
    public Endereco(String rua, String numero, String complemento,
                    String bairro, String cidade, String estado, String cep) {
        this.rua = rua;
        this.numero = numero;
        this.complemento = complemento;
        this.bairro = bairro;
        this.cidade = cidade;
        this.estado = estado;
        this.cep = cep;
    }
    
    public String formatarCompleto() {
        StringBuilder sb = new StringBuilder(rua + ", " + numero);
        if (complemento != null) {
            sb.append(" - ").append(complemento);
        }
        sb.append(" - ").append(bairro);
        sb.append(" - ").append(cidade).append("/").append(estado);
        sb.append(" - CEP: ").append(cep);
        return sb.toString();
    }
}

// Classe para valores monetários
public class ValoresPedido {
    private final double total;
    private final double desconto;
    private final double frete;
    private final double imposto;
    
    public double getSubtotal() {
        return total - desconto;
    }
    
    public double getTotalFinal() {
        return getSubtotal() + frete + imposto;
    }
}

// Uso limpo
public class RelatorioService {
    
    public void gerarRelatorio(Cliente cliente, Endereco endereco, 
                                ValoresPedido valores) {
        System.out.println("Cliente: " + cliente.getNome());
        System.out.println("Email: " + cliente.getEmail());
        System.out.println("Endereço: " + endereco.formatarCompleto());
        System.out.println("Total: R$ " + valores.getTotalFinal());
    }
}

🔧 Técnicas de Refatoração

Extract Class (Extrair Classe)

Crie uma nova classe para conter os dados que estão sempre juntos. Mova os campos e crie métodos úteis para manipular esses dados.

Introduce Parameter Object

Substitua parâmetros que aparecem juntos por um objeto que os contenha. Reduz a assinatura dos métodos.

Preserve Whole Object

Ao invés de passar valores individuais, passe o objeto inteiro que os contém.

Value Object Pattern

Crie objetos de valor imutáveis para representar conceitos do domínio. Adicione métodos de validação e formatação.
