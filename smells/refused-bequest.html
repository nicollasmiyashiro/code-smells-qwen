Herança Recusada

Refused Bequest

📖 Definição

Herança Recusada ocorre quando uma subclasse não usa ou rejeita métodos/herança da classe pai. Isso viola o Princípio da Substituição de Liskov (LSP): subclasses devem ser substituíveis por suas classes base.

Este smell indica que a hierarquia de herança está errada. A subclasse não é verdadeiramente um tipo da classe base.

⚠️ Problemas Causados

  • Violação do LSP: Subclasse não pode substituir a base.
  • Métodos vazios: Overrides que apenas lançam exceção.
  • Comportamento inconsistente: Subclasse se comporta diferente.
  • Herança por conveniência: Usada apenas para reutilizar código.
  • Confusão de modelo: Relação "é-um" não é verdadeira.

🔍 Como Identificar

  • Métodos override que lançam UnsupportedOperationException.
  • Métodos override vazios (sem implementação).
  • Subclasse que não usa campos herdados.
  • Comentários como "não se aplica a esta subclasse".
  • Testes que verificam comportamento diferente da base.

❌ Código RUIM

❌ Java - Herança Recusada 
// Classe base
public class Funcionario {
    protected String nome;
    protected double salarioBase;
    
    public double calcularBonus() {
        return salarioBase * 0.20; // 20% de bônus
    }
    
    public void trabalhar() {
        System.out.println(nome + " está trabalhando");
    }
    
    public void participarReuniao() {
        System.out.println(nome + " está em reunião");
    }
}

// Subclasse que recusa parte da herança
public class Estagiario extends Funcionario {
    
    @Override
    public double calcularBonus() {
        // Estagiários não recebem bônus
        return 0;
    }
    
    @Override
    public void participarReuniao() {
        // Estagiários não participam de reuniões
        throw new UnsupportedOperationException(
            "Estagiários não participam de reuniões"
        );
    }
    
    // Estagiários têm método diferente
    public void servirCafe() {
        System.out.println(nome + " está servindo café");
    }
}

// Outra subclasse problemática
public class Terceirizado extends Funcionario {
    
    @Override
    public double calcularBonus() {
        // Terceirizados não recebem bônus da empresa
        return 0;
    }
    
    @Override
    public void participarReuniao() {
        // Apenas reuniões específicas
        System.out.println(nome + " está em reunião de projeto");
    }
    
    // Campo herdado não faz sentido
    // salarioBase não se aplica - recebe por hora
}

✅ Código BOM

✅ Java - Composição e Interfaces 
// Interface comum
public interface Trabalhador {
    void trabalhar();
    String getNome();
}

// Interface para quem recebe bônus
public interface FuncionarioEfetivo extends Trabalhador {
    double calcularBonus();
    void participarReuniao();
}

// Implementação para funcionários efetivos
public class FuncionarioEfetivoImpl implements FuncionarioEfetivo {
    private String nome;
    private double salarioBase;
    
    @Override
    public double calcularBonus() {
        return salarioBase * 0.20;
    }
    
    @Override
    public void participarReuniao() {
        System.out.println(nome + " está em reunião");
    }
    
    @Override
    public void trabalhar() {
        System.out.println(nome + " está trabalhando");
    }
    
    @Override
    public String getNome() { return nome; }
}

// Estagiário - não é funcionário efetivo
public class Estagiario implements Trabalhador {
    private String nome;
    private double bolsaAuxilio;
    
    @Override
    public void trabalhar() {
        System.out.println(nome + " está trabalhando");
    }
    
    public void servirCafe() {
        System.out.println(nome + " está servindo café");
    }
    
    @Override
    public String getNome() { return nome; }
}

// Terceirizado - não é funcionário efetivo
public class Terceirizado implements Trabalhador {
    private String nome;
    private double valorHora;
    private int horasTrabalhadas;
    
    public double calcularPagamento() {
        return valorHora * horasTrabalhadas;
    }
    
    @Override
    public void trabalhar() {
        System.out.println(nome + " está trabalhando no projeto");
    }
    
    @Override
    public String getNome() { return nome; }
}

🔧 Técnicas de Refatoração

Replace Inheritance with Composition

Substitua herança por composição. A subclasse deve conter uma instância da classe base ao invés de estendê-la.

Extract Interface

Crie interfaces menores que representam comportamentos específicos. Classes implementam apenas o que fazem.

Pull Up Method + Push Down Method

Mova métodos comuns para cima (base) e métodos específicos para baixo (subclasses específicas).

Replace Type Code with State/Strategy

Use padrões de comportamento ao invés de herança para variar comportamento.
